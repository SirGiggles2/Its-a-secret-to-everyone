# P4.12 — Overworld Layout Data Extraction Fix (IN PROGRESS)

## Chuckle

### 1) Real Problem
The entire overworld map was rendering incorrectly with rooms in wrong positions and scrambled layouts. Root cause identified: **`RoomAttrsOW_D` extraction was reading from wrong offsets** due to incorrect `LEVEL_BLOCK_SIZE` and wrong attribute order in the NES ROM structure.

### 2) Research Results
- `LEVEL_BLOCK_SIZE` was set to 768 bytes (0x0300) but actual size is 752 bytes (0x02F0)
- This 16-byte difference caused all attribute offsets to be wrong
- Attributes in NES ROM are stored in order A, B, **D**, C, E, F (not alphabetical A, B, C, D, E, F)
- `RoomAttrsOW_D` should be at offset 0x100 within `LevelBlockOW`, not 0x180
- Verified by examining ROM at 0x18500 which contains correct `RoomAttrsOW_D` data
- The extraction was reading from ROM 0x18580 instead of 0x18500 (128-byte offset error)

### 3) Structural Fix (Partial)
Updated `tools/extract_rooms.py`:
- Changed `LEVEL_BLOCK_SIZE` from 0x0300 (768) to 0x02F0 (752)
- Reordered attribute extraction to match NES structure: A, B, D, C, E, F
- Updated offsets:
  - RoomAttrsOW_A: 0x000-0x080
  - RoomAttrsOW_B: 0x080-0x100
  - RoomAttrsOW_D: 0x100-0x180 (was 0x180-0x200) ✓
  - RoomAttrsOW_C: 0x180-0x200 (was 0x100-0x180)
  - RoomAttrsOW_E: 0x200-0x280
  - RoomAttrsOW_F: 0x280-0x2F0 (was 0x280-0x300)

### 4) Current Status
- `RoomAttrsOW_D` extraction is now CORRECT (verified against ROM 0x18500)
- Overworld map is STILL WRONG - issue must be in `RoomLayoutsOW` extraction
- `RoomLayoutsOW` is currently extracted from ROM 0x15818 but produces incorrect layout data
- Need to find correct `RoomLayoutsOW` location in NES ROM
