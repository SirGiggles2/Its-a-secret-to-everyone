# P4.11 — Overworld ROM Layout Parity (Edge Transition Wrap)

## Chuckle

### 1) Real Problem
Overworld traversal could diverge from NES room adjacency because edge transitions were using border clamps in scene logic instead of NES-style wrapped room-id arithmetic.

### 2) Research Results
- NES `ScrollWorldH` / transition flow in `reference/aldonunez/Z_05.asm` updates room progression using increment/decrement behavior and does not apply hard border-clamp checks in transition arithmetic.
- Current implementation in `src/scenes/zelda_data_smoke.asm` had explicit left/right/top/bottom clamp branches that prevented wrapped room-id progression at edges.

### 3) Structural Fix
Updated transition target-room math in `Scene_ZeldaDataSmoke_HandleMovement`:
- Left: `D3 = (CURRENT_ROOM_ID - 1) & $7F`
- Right: `D3 = (CURRENT_ROOM_ID + 1) & $7F`
- Up: `D3 = (CURRENT_ROOM_ID - $10) & $7F`
- Down: `D3 = (CURRENT_ROOM_ID + $10) & $7F`

Removed clamp-only branches (`.clamp_left`, `.clamp_right`, `.clamp_top`, `.clamp_bottom`) so transitions follow NES-style wrapped room-id behavior.
