# WHAT IF

Exact port of The Legend of Zelda (NES, 1986) to the Sega Genesis / Mega Drive.

Native 68000 assembly. Real Genesis hardware target. 100% gameplay accuracy.

## Structure

```
src/                    68000 assembly source (vasm, Motorola syntax)
  main.asm              ROM vectors, header, frame loop
  platform.asm          Genesis hardware init, VDP, joypad
  renderer.asm          VRAM/CRAM primitives (to be extended)
  scenes/               Hardware test scenes
  data/                 Extracted game data (generated by Python)
reference/aldonunez/    NES 6502 disassembly (behavior reference)
tools/                  Build tools and data extraction scripts
builds/                 Output ROM images and listings
docs/                   Design docs and reference maps
```

## Build

```bat
build.bat
```

Requires `vasmm68k_mot.exe` and Python 3.12+.

## Approach

1. Extract all game data (tiles, rooms, enemies, items, audio) from the NES ROM using Python scripts
2. Build a compact Genesis engine in 68000 ASM that faithfully executes that data
3. Each implementation phase produces a testable ROM

## Phases

| Phase | What | Test |
|-------|------|------|
| 1 | Data extraction (Python) | Real Zelda tiles in VRAM viewer |
| 2 | Renderer upgrade (DMA, sprites, scroll) | Room renders with HUD |
| 3 | Room system | Starting room loads, transitions work |
| 4 | Player system | NES-accurate Link movement and combat |
| 5 | Object/enemy system | Enemies walk, fight, drop items |
| 6 | Mode manager | Full game flow (title to gameplay to death) |
| 7 | Frontend & menus | Title, file select, inventory |
| 8 | Audio engine | Music and SFX |
| 9 | Polish & second quest | Full playthrough both quests |
